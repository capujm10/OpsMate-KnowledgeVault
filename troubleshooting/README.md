# Troubleshooting

This directory contains diagnostic workflows and decision trees.

Troubleshooting documents should guide investigation from symptoms to evidence to likely cause. They should make branching logic explicit and keep destructive remediation behind review gates.

## Recommended Flow

1. Define the symptom.
2. Collect read-only evidence.
3. Separate user impact from system health.
4. Identify the failing boundary.
5. Confirm with telemetry.
6. Select the safest reversible action.
7. Validate recovery.
8. Capture learning in a knowledge entry or RCA.
