# Knowledge

This directory contains curated operational knowledge entries and recurring failure patterns.

Use it for durable investigation knowledge that should be reusable across incidents, support escalations, troubleshooting sessions, or AI-assisted retrieval.

## Primary Subareas

- `failure-patterns/`: recurring operational failure modes and symptom clusters.
- `platforms/`: platform-specific troubleshooting knowledge.
- `domains/`: domain-specific knowledge such as TLS, API integration, queues, authentication, and performance.

## Entry Quality Bar

Each substantial entry should answer:

- What does this failure look like?
- What signals confirm it?
- What signals can mislead the investigator?
- What evidence should be collected first?
- What is the likely root cause range?
- What remediation is safe, reversible, and validated?
- What related runbooks, RCA examples, or telemetry guides should be linked?

Prefer concise and actionable entries over broad essays.
