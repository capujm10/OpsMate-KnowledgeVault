# Failure Pattern Catalog

Failure patterns describe recurring operational problems in a compact, searchable format.

They should not read like incident stories. They should describe a reusable pattern:

- Common symptoms
- Triage signals
- Likely causes
- Fast checks
- Safe remediation paths
- Validation steps
- Related runbooks and RCA examples

Use [../../templates/failure-pattern.md](../../templates/failure-pattern.md) for new entries.
